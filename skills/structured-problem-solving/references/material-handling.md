# Material Handling

资料处理是结构化分析的基础。用户提供文档、长文本、网页、表格、代码、访谈记录或其他外部材料时，先盘点材料覆盖范围、可信度和缺口，再进入深度分析。

资料盘点不等于必须落盘。Quick 和 Standard 模式默认在对话中引用附件、链接或消息内容并完成盘点；只有进入 Project 模式后，才按下述目录归档。Project 模式默认使用轻量结构，来源较多、周期较长或溯源要求较高时再扩展为完整资料库。

## `0-inputs/` 与 `0-materials/`

仅在 Project 模式中使用以下目录。

`0-inputs/` 放原始资料，原则是不改动、可追溯、作为证据源头。

`0-materials/` 放处理后、可用于分析的资料，是从 `0-inputs/` 派生出来的工作材料。

```text
0-inputs/     = 原始材料，不动它
0-materials/  = 分析材料，可以整理、转换、摘录、标注
```

## 资料处理流程

1. **选择处理载体**
   - Quick / Standard 模式：在对话中列出材料名称或位置、来源、日期、可信度和用途，不创建目录或复制附件。
   - Project 模式：将需要持续引用的原始文件、粘贴文本、链接摘录或访谈记录保存到 `0-inputs/`。
   - 不改写原始资料；需要清洗且已进入 Project 模式时，另存到 `0-materials/`。

2. **转换为可分析格式**
   - 只有现有工具无法直接可靠读取，或需要持久化可检索副本时才转换。
   - PDF / Word / Excel / PowerPoint / EPUB / HTML / 字幕 / URL 等资料，优先调用 `markdown-conversion` skill 转成 Markdown。
   - Project 模式将转换结果放入 `0-materials/converted/`；资料已经是可读 Markdown 或纯文本时，可直接放入 `0-materials/`。

3. **建立来源索引**
   - 轻量模式在 `0-materials/evidence.md` 的 `Sources` 区记录资料名称、来源、日期、可信度、用途。
   - 完整模式在 `0-materials/source-index.md` 记录资料名称、来源、日期、可信度、用途。
   - 对关键事实保留来源位置，例如文件名、章节、页码、URL 或访谈对象。

4. **提取证据**
   - 在 `0-materials/evidence.md` 中整理关键事实、数字、引用、反常点和不确定信息。
   - 每条证据都标明来源；无法溯源的信息只能作为假设或待验证项。

5. **区分事实、假设和观点**
   - 事实：有来源或可验证的数据。
   - 假设：基于有限信息的可验证猜测。
   - 观点：主观判断或解释，需要后续证据支持。

6. **形成资料盘点**
   - 轻量模式在 `evidence.md` 中保留 `Sources` 和 `Gaps` 区即可。
   - 完整模式在 `0-materials/material-inventory.md` 中列出已获取资料、已处理资料、可信度、覆盖的问题范围。
   - 完整模式在 `0-materials/gaps.md` 中列出继续分析所需但尚未获得的资料。
   - 如果资料缺口会影响问题定义，先输出临时定义并标注“待资料补充后复核”。

## 落盘输出模式

如果只需在对话中完成分析，直接输出资料来源、事实、假设、观点和缺口，不创建以下文件。

### 轻量模式

适用条件：

- 来源为 1-3 个。
- 不需要长周期协作。
- 资料主要用于一次性问题定义、判断或建议。

只维护一个分析文件；仅在确有原始资料时创建 `0-inputs/`：

```text
0-materials/
└── evidence.md             # 来源、关键事实、假设、观点、资料缺口
```

`evidence.md` 使用以下结构：

```markdown
# Evidence

## Sources
- [资料名] - [来源 / 日期 / 可信度 / 用途]

## Facts
- [事实]；来源：[文件 / 页码 / URL / 访谈对象]

## Assumptions
- [可验证猜测]；验证方式：[...]

## Opinions
- [主观判断]；依据或风险：[...]

## Gaps
- [缺口]；影响：[问题定义 / 拆解 / 优先级 / 建议]
```

### 完整模式

适用条件：

- 来源不少于 4 个。
- 涉及多轮调研、多人协作或高风险决策。
- 需要严格追踪来源、证据和资料缺口。

```text
0-materials/
├── converted/              # 转换后的 Markdown 或文本
├── domain-glossary.md      # 可选：关键术语存在歧义时创建
├── material-inventory.md   # 已获取资料盘点
├── source-index.md         # 来源索引
├── evidence.md             # 证据清单
└── gaps.md                 # 资料缺口和待验证问题
```

进入结构化分析前，至少要能回答：我们已经知道什么、还不知道什么、哪些信息只是猜测。轻量模式能回答即可，不要强制拆文件。

## 后续资料补充机制

当后续分析发现需要新增资料时，不要把新资料直接写进分析结论。按以下流程回填：

1. **登记缺口**
   - 对话模式在当前回答中记录缺口及下一步验证动作。
   - 轻量落盘模式在 `0-materials/evidence.md` 的 `Gaps` 区记录；完整模式在 `0-materials/gaps.md` 记录。
   - 标注缺口类型：事实缺口、数据缺口、专家判断缺口、用户偏好缺口、外部约束缺口。

2. **补充原始资料**
   - 对话模式继续引用新附件、链接或用户补充信息，不为此自动切换到 Project 模式。
   - Project 模式将新获得的文件、访谈、网页摘录或用户补充信息先放入 `0-inputs/`。
   - 文件名建议带日期或来源，例如 `20260630-user-interview.md`、`20260630-market-report.pdf`。

3. **处理并索引**
   - 转换或清洗后放入 `0-materials/converted/` 或 `0-materials/`。
   - 轻量模式更新 `evidence.md`。
   - 完整模式更新 `source-index.md`、`material-inventory.md`、`evidence.md`。
   - 如果缺口已解决，轻量模式在 `evidence.md` 更新；完整模式在 `gaps.md` 中标记状态为 `已补充`，并写明来源。

4. **回到受影响阶段**
   - 对话模式重新执行受影响的分析步骤并说明结论变化。
   - Project 模式优先更新已有阶段文件；如果尚无对应文件，先更新 `brief.md`，只有新资料产生了需要长期保留的实质分析产物时才创建阶段目录。
   - 如果新资料改变问题理解，更新 `1-problem-definition/`。
   - 如果新资料改变拆解方式，更新 `2-problem-structuring/`。
   - 如果新资料改变优先级，更新 `3-prioritization/`。
   - 如果新资料验证或推翻假设，更新 `5-analysis/` 和 `6-synthesis/`。

5. **更新总控文件**
   - 仅在 Project 模式中更新 `brief.md` 的“资料状态”“证据状态”和“下一步”。

资料补充是循环过程。允许阶段回退；不要为了保持流程线性而忽略新证据。
