# McKinsey Seven-Step Workflow

本技能以麦肯锡七步问题解决法作为主流程。Dalio 的“目标 -> 问题 -> 诊断 -> 设计 -> 执行”作为每一步中的循环心智模型：先明确目标，发现阻碍，诊断根因，设计方案，再执行并复盘。

## Step 1: 明确问题

目标是确认“到底在解决什么问题”，避免把精力浪费在表面请求上。

在明确问题前，先完成资料盘点：

- 已获取哪些资料？
- 这些资料覆盖了问题的哪些部分？
- 哪些关键资料仍缺失？
- 缺失资料是否会影响问题定义？

如果资料不足以完成最终定义，输出“临时问题定义”，并列出必须补充的资料。

问题定义工具箱：

| 要素 | 要回答的问题 |
|---|---|
| 背景 | 为什么现在要解决？发生了什么变化？ |
| 成功标准 | 什么结果算解决？如何判断好坏？ |
| 边界 | 哪些包含在内？哪些明确不做？ |
| 限制条件 | 时间、预算、政策、技术、人力等限制是什么？ |
| 相关人 | 谁决策、谁执行、谁受影响、谁提供信息？ |
| 资源 | 已有数据、资料、工具、专家、时间和预算是什么？ |

判断标准：正确问题被解决后，它引发的表象问题应随之消失。

## Step 2: 分解问题

用“切”把复杂问题拆成可分析单元。优先保证 MECE：相互独立、完全穷尽。

| 方法 | 适用场景 | 示例 |
|---|---|---|
| 公式法 | 有明确公式或指标关系 | 利润 = 收入 - 成本 |
| 子目录列举法 | 可按类别穷尽拆分 | 成本 = 固定成本 + 可变成本 |
| 流程法 | 有生命周期或操作流程 | 研发、生产、销售、售后 |
| 逻辑框架法 | 可用对立面或通用框架 | 内部/外部，供给/需求 |

复杂问题尽量使用“3-3 原则”：至少 3 个维度、至少 3 层拆解。需要找根因时可用鱼骨图；单维列表不足以产生洞见时，使用二维或三维图谱，例如“战略重要性 x 执行难度 x 潜在收益”。

## Step 3: 优先排序

不是所有子问题都值得同等投入。优先排序用于找到最值得先解决的关键杠杆。

- **80/20 法则** - 找到对结果影响最大的少数因素。
- **重要性-紧急性矩阵** - 区分立刻处理、计划处理、授权处理和暂缓处理。
- **影响-难度矩阵** - 优先处理高影响、低难度项；谨慎评估高影响、高难度项。

输出应包括：优先级、理由、暂不处理项和取舍依据。

## Step 4: 工作计划

将分析转化为可执行计划，避免停留在框架层面。

工作计划只管理已经决定要执行的任务。尚未回答的研究、原型、追问或前置动作，如果会影响路线判断，应先放在 `decision-map.md` 的 `Open Questions` 中。

工作计划必须包含：

- 分析任务和待回答问题。
- 时间表和阶段里程碑。
- 责任人或执行角色。
- 所需资料、工具和资源。
- 风险、依赖和应对方案。

使用 SMART 原则检查计划：具体、可衡量、可实现、相关、有时限。

## Step 5: 分析论证

基于结构化分析形成“第一天的答案”，再用数据和证据验证。假设不是最终结论，而是分析论证的标靶。

合格假设必须：

- 能回答原问题。
- 可被证据支持或推翻。
- 指向明确的验证动作。
- 明确区分事实、判断和待验证猜测。

验证方式：

- 案头调研：行业报告、公开资料、历史数据、竞品信息。
- 实地调研：访谈、问卷、观察、实验、一手数据。
- 反例检查：寻找能推翻当前假设的情况。
- 常识估算：用粗略计算检查数字是否可信。

警惕逻辑陷阱：相关关系不等于因果关系；归纳不能替代严格证明；单一视角容易遗漏关键维度；选择性数据和偷换概念会制造假洞见。

## Step 6: 综合建议

把分析结果综合成可选择、可比较、可执行的方案。

综合建议必须回答：

- 主要结论是什么？
- 可选方案有哪些？
- 每个方案的收益、成本、风险和依赖是什么？
- 推荐哪个方案，为什么？
- 哪些假设如果被推翻，会改变建议？

## Step 7: 方案呈现

交付要帮助听众决策或行动。默认结论先行，再给支撑逻辑和证据。

- **金字塔原则** - 结论 -> 关键论点 -> 支撑证据。
- **3W2H 故事线** - 为什么、用什么、怎么做、为什么是你、投入产出。
- **SCP+I** - 行业结构、玩家行为、业绩表现、外部冲击，用于讲清“为什么现在有机会或问题”。

交付前必须检查听众是谁、他们关心什么、他们需要做什么决策。

## Structured Analysis Template

```markdown
## 问题陈述
[一句话定义问题]

## 背景与目标
[为什么要解决，以及希望达成什么]

## 成功标准
- [可验证标准]

## 边界与限制
- 范围内：[...]
- 范围外：[...]
- 限制条件：[...] 

## 结构化拆解
[MECE 逻辑树或多维图谱]

## 优先排序
- [关键子问题 / 优先级 / 理由 / 暂不处理项]

## 工作计划
- [任务 / 时间 / 责任人或角色 / 资源 / 风险]

## 分析论证
- 假设 1：[内容]；依据：[已有证据]；待验证：[缺口]
- 假设 2：[内容]；依据：[已有证据]；待验证：[缺口]

## 验证与证据
- [证据来源、验证方法、反例检查]

## 综合建议
[方案比较、推荐方案、风险和取舍]

## 方案呈现 / 下一步
[当前最合理的行动建议]
```
